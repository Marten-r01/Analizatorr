# Analizator
pet-project combine dna and program
финальный проект на стыке FullStack разработки на Kotline и генной инженирии

Краткое описание:
Система представляет собой Full Stack приложение для загрузки, хранения и анализа генетических последовательностей в формате FASTA. Пользователь загружает файл, система выполняет его валидацию, парсинг и биоинформатический анализ, после чего отображает результаты в удобном визуальном виде. Проект предназначен для учебного и демонстрационного использования в области биоинформатики и full stack разработки.

Актуальность

Проект актуален, поскольку задачи хранения, обработки и анализа генетических последовательностей имеют высокую практическую значимость в биоинформатике и образовании. Разработка сервиса позволяет автоматизировать первичный анализ FASTA-файлов, уменьшить количество ручных операций и обеспечить наглядное представление результатов.

Востребованность

Проект востребован как учебный и прикладной инструмент для студентов, преподавателей и начинающих специалистов, работающих с генетическими данными. Сервис позволяет быстро выполнять базовый анализ последовательностей, сохранять результаты и визуализировать их в удобной форме.

Архитектура и стек

Общая схема:
Система состоит из frontend-клиента и backend-сервера.
Frontend отвечает за загрузку файлов, отображение последовательности и визуализацию результатов анализа.
Backend принимает файлы, валидирует входные данные, парсит FASTA, выполняет анализ, сохраняет результаты и возвращает их клиенту через REST API.
Отдельные микросервисы в базовой версии не предусмотрены.

Backend:

Язык/фреймворк: Kotlin 2.x, Ktor
БД: PostgreSQL
Доступ к БД: Exposed
Взаимодействие: REST API, HTTP, JSON, Multipart/form-data
Сериализация: kotlinx.serialization

Frontend / клиент:

Тип: Web-клиент на Compose Multiplatform (Web/WASM)
Взаимодействие с backend: REST API по HTTP
Основные функции: загрузка файла, визуализация последовательности, вывод результатов анализа, построение графиков

Доп. микросервисы:

В рамках текущего проекта отдельные микросервисы не выделяются
Логика парсинга FASTA, анализа последовательности и сохранения данных реализуется внутри backend-приложения как отдельные модули
