# Analizator

**Pet-project на стыке FullStack-разработки на Kotlin и биоинформатики**

Analizator — это Full Stack приложение для загрузки, хранения и анализа генетических последовательностей в формате **FASTA**.  
Пользователь загружает файл, система выполняет его валидацию, парсинг и базовый биоинформатический анализ, после чего отображает результаты в удобном визуальном виде.

Проект предназначен для учебного и демонстрационного использования в области:
- биоинформатики
- full stack разработки
- алгоритмической обработки предметно-ориентированных данных

## О проекте

Система решает задачу первичного анализа нуклеотидных последовательностей.  
После загрузки FASTA-файла backend выполняет:
- валидацию входных данных
- разбор FASTA-структуры
- нормализацию последовательности
- вычисление GC-состава
- поиск ORF
- трансляцию ДНК в белок
- сохранение результатов анализа

Frontend предоставляет интерфейс для:
- загрузки файла
- отображения последовательности
- визуализации результатов анализа
- вывода статистики и графиков

## Актуальность

Проект актуален, поскольку задачи хранения, обработки и анализа генетических последовательностей имеют высокую практическую значимость в биоинформатике и образовании.

Разработка такого сервиса позволяет:
- автоматизировать первичный анализ FASTA-файлов
- сократить количество ручных операций
- уменьшить вероятность ошибок при обработке данных
- сделать результаты анализа наглядными и удобными для интерпретации

Кроме прикладной ценности, проект интересен как инженерная задача, объединяющая:
- backend-разработку
- frontend-разработку
- проектирование БД
- алгоритмы обработки строк и последовательностей
- доменную логику из биоинформатики

## Архитектура

Система состоит из двух основных частей:

### Frontend
Отвечает за:
- загрузку файлов
- отображение последовательности
- визуализацию результатов анализа
- построение графиков

### Backend
Отвечает за:
- приём файлов через HTTP
- валидацию входных данных
- парсинг FASTA
- биоинформатический анализ
- сохранение результатов
- REST API для frontend-клиента

В базовой версии отдельные микросервисы не выделяются.  
Логика парсинга, анализа и сохранения данных реализуется внутри backend-приложения как отдельные модули.
